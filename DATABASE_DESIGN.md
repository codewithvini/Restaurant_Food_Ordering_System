# Database Design Document
## Restaurant Food Ordering System

---

## Database Overview

**Database Name:** `restaurant_system`
**Database Type:** MongoDB (NoSQL)
**Connection:** MongoDB Atlas (Cloud)

---

## Collections Schema

### 1. Menu Collection (`menu`)

Stores restaurant menu items with details about food items available for ordering.

**Schema Structure:**
```json
{
  "_id": ObjectId("..."),
  "name": String,
  "category": String,
  "price": Number,
  "description": String,
  "availability": Boolean,
  "created_at": Date,
  "updated_at": Date (optional)
}
```

**Field Details:**
- `_id`: Unique identifier (automatically generated by MongoDB)
- `name`: Name of the menu item (e.g., "Margherita Pizza")
- `category`: Category of item (Appetizer, Main Course, Dessert, Beverage, Side Dish)
- `price`: Price in currency (₹)
- `description`: Detailed description of the item
- `availability`: Boolean indicating if item is currently available
- `created_at`: Timestamp when item was added
- `updated_at`: Timestamp when item was last modified

**Sample Document:**
```json
{
  "_id": ObjectId("65f1234567890abcdef12345"),
  "name": "Margherita Pizza",
  "category": "Main Course",
  "price": 299.0,
  "description": "Classic pizza with tomato sauce, mozzarella, and basil",
  "availability": true,
  "created_at": ISODate("2024-03-13T10:30:00Z")
}
```

**Indexes:**
- Primary index on `_id`
- Text index on `name` and `category` for search functionality

---

### 2. Orders Collection (`orders`)

Stores customer orders with order details and status tracking.

**Schema Structure:**
```json
{
  "_id": ObjectId("..."),
  "customer_name": String,
  "phone": String,
  "items": Array[String],
  "total_amount": Number,
  "payment_method": String,
  "status": String,
  "order_date": Date,
  "updated_at": Date (optional)
}
```

**Field Details:**
- `_id`: Unique order identifier
- `customer_name`: Name of the customer placing the order
- `phone`: Customer contact number
- `items`: Array of ordered item names
- `total_amount`: Total order amount in currency
- `payment_method`: Payment method (Cash, Card, UPI, Online)
- `status`: Order status (Pending, Preparing, Ready, Delivered, Cancelled)
- `order_date`: Timestamp when order was created
- `updated_at`: Timestamp when order status was last updated

**Sample Document:**
```json
{
  "_id": ObjectId("65f1234567890abcdef12346"),
  "customer_name": "Rahul Sharma",
  "phone": "9876543210",
  "items": ["Margherita Pizza", "Cold Coffee"],
  "total_amount": 418.0,
  "payment_method": "UPI",
  "status": "Delivered",
  "order_date": ISODate("2024-03-13T11:15:00Z"),
  "updated_at": ISODate("2024-03-13T12:00:00Z")
}
```

**Indexes:**
- Primary index on `_id`
- Index on `customer_name` and `phone` for search
- Index on `order_date` for sorting (descending)

---

### 3. Feedback Collection (`feedback`)

Stores customer feedback and ratings for the restaurant.

**Schema Structure:**
```json
{
  "_id": ObjectId("..."),
  "customer_name": String,
  "email": String,
  "rating": Number,
  "comments": String,
  "feedback_date": Date
}
```

**Field Details:**
- `_id`: Unique feedback identifier
- `customer_name`: Name of the customer providing feedback
- `email`: Customer email address
- `rating`: Rating on scale of 1-5
- `comments`: Detailed feedback comments
- `feedback_date`: Timestamp when feedback was submitted

**Sample Document:**
```json
{
  "_id": ObjectId("65f1234567890abcdef12347"),
  "customer_name": "Priya Patel",
  "email": "priya@example.com",
  "rating": 5,
  "comments": "Excellent food and service! The pizza was amazing.",
  "feedback_date": ISODate("2024-03-13T13:30:00Z")
}
```

**Indexes:**
- Primary index on `_id`
- Index on `feedback_date` for sorting (descending)
- Index on `rating` for aggregation queries

## Aggregation Pipelines

### Average Rating Calculation
Used to calculate the average rating and total feedback count:

```javascript
db.feedback.aggregate([
  {
    $group: {
      _id: null,
      average_rating: { $avg: "$rating" },
      total_feedback: { $sum: 1 }
    }
  }
])
```

## CRUD Operations Summary

### Menu Management
- **Create**: Add new menu items
- **Read**: View all menu items, search by name/category
- **Update**: Modify item details, price, availability
- **Delete**: Remove items from menu

### Order Tracking
- **Create**: Place new orders
- **Read**: View all orders, search by customer/phone
- **Update**: Change order status
- **Delete**: Remove orders (admin function)

### Customer Feedback
- **Create**: Submit customer feedback
- **Read**: View all feedback, calculate average rating
- **Update**: Not implemented (feedback is read-only once submitted)
- **Delete**: Remove feedback (admin function)

---

## Data Relationships

Although MongoDB is a NoSQL database and doesn't enforce foreign key relationships, the application maintains logical relationships:

1. **Menu → Orders**: Order items reference menu item names
2. **Orders → Customers**: Orders are linked to customers via name and phone
3. **Feedback → Customers**: Feedback is associated with customer details

---

## Security Considerations

1. **Authentication**: MongoDB Atlas connection requires username and password
2. **Connection String**: Stored in `config.py` (should be kept secure and not committed to version control)
3. **Data Validation**: Application-level validation for all inputs
4. **Error Handling**: Comprehensive error handling for database operations

---

## Advantages of MongoDB for This System

1. **Flexible Schema**: Easy to add new fields without migrations
2. **Document Model**: Natural representation of menu items, orders, and feedback
3. **Scalability**: MongoDB Atlas provides automatic scaling
4. **JSON Format**: Easy integration with Python using pymongo
5. **Aggregation Framework**: Powerful queries for analytics (e.g., average ratings)
6. **Cloud-Based**: MongoDB Atlas provides managed database service

**Driver:** pymongo
**Connection String Format:**
```
mongodb+srv://<username>:<password>@<cluster>.mongodb.net/<database>?retryWrites=true&w=majority
```

- Database User: Create user with read/write privileges
